프로젝트의 가이드 문서 작성은 하기의 규칙을 따름.

- 공통 기능(shared, core)을 수정하는 경우 작업완료 시 가이드 문서를 업데이트.
- apps/ 의 자식 폴더들과 packages/ 의 자식 폴더들은 서브 루트로 칭함.
- 루트와 서브 루트에만 docs/ 폴더를 만들어서 가이드 문서를 관리함.
- README.md 파일은 필요한 모든 폴더에 존재할 수 있음.
- 상세 가이드 문서는 /docs 폴더에 정리.
- README.md는 사용법 위주로 간략히 정리하고 개념 등의 상세한 설명은 링크(/docs 의 문서나 웹 링크)로 대체.
- 공통 기능(shared, core)을 수정하는 경우 작업완료 시 가이드 문서를 업데이트.
- 가이드 문서들은 현재 코드 기준으로 현행화가 필요함.
- 문서들 중에 중복되는 문서는 합치고, 필요 없는 문서는 제거.
- 작업 이력은 필요 없고 사용법 위주로 정리.
